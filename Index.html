Frequently Asked Questions

HubSpot deals sync automatically when you enter a JLID in the Communication Center or Learner History page. Use the Fetch button to trigger a manual pull. Data refreshes include learner name, status, current teacher, course, and deal value.
All failed emails are logged in the Email Activities page with the error reason. You can retry by reopening the Communication Center for that learner. Migration emails log separately in the Audit Log.
Scores are computed in the Persona & Replacement Engine using: course proficiency (must be ≥71%), slot availability on the requested date, audit grade (last 45 days), trait match, age-group compatibility, and current load. Each factor is weighted to produce a 0–100% score.
Migrations create HubSpot tickets and send email notifications — these cannot be automatically undone. To reassign a learner, initiate a new migration through the Communication Center using the original teacher as the new teacher. Contact the team if emails need to be recalled.
Availability is read from the teacher's Google Calendar. Create events titled "Availability" or "Teaching Hours" on the teacher's calendar for the relevant slots. The Persona Engine reads these automatically when matching teachers for a requested date.
Grades are based on total audit score: A = ≥65 (Excellent), B = 50–64 (Good), C = 35–49 (Needs Improvement), D = <35 (Critical). An EWS (Early Warning Status) flag is raised for D grades or red flags.
A Red Flag is a specific concern raised during an audit (e.g. "Learner disengaged", "Not following curriculum"). When new red flags appear in any audit sheet, an automated email alert is sent to the teacher's TP Manager daily at 8 AM. Alerts are de-duplicated — the same flag won't re-trigger.
Open the Audit Center → TP Dashboard tab → click the Export CSV button in the table toolbar. This downloads all visible teacher rows including risk score, grade, and audit date as a CSV file.
